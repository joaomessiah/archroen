# Roman-villa set — frozen reports & outputs

The **real-world application** corpus: the South-Limburg Roman-villa reports the workflow was run on.
This folder keeps a **frozen copy of the source report PDFs and the pipeline outputs** so the thesis
results are preserved even after the live files are regenerated.

| Folder | What's inside |
|---|---|
| [input_reports/](input_reports/) | The source report PDFs, one per report id (e.g. `12703.pdf`). |
| [outputs/](outputs/) | One `<report_id>.csv` pottery summary per report. |

This set has **no gold standards** (it is the application, not the accuracy measurement). A live copy
of the PDFs is under `input_files/reports/south_limburg_villas/`. For the full picture, see the
[datasets overview](../README.md) and [../roman_villas.md](../roman_villas.md).
